# Adam Ellahi Design System

This folder is the working asset library for Project F1.

It supports the human-readable Bible in:

`Project_F1_Design_Bible_v1`

Start with the Formula One Digital Design Language:

`Project_F1_Design_Bible_v1/00_Executive_Handoff/FORMULA_ONE_DIGITAL_DESIGN_LANGUAGE_F1DL.md`

## Contents

- `01 Brand` - brand rules and future brand exports.
- `02 Logos` - SVG logo system.
- `03 Favicons` - favicon and app icon SVG starters.
- `04 Colours` - colour references.
- `05 Typography` - typography references.
- `06 Icons` - motorsport icon system.
- `07 Components` - component specifications.
- `08 Desktop UI` - future desktop UI exports.
- `09 Mobile UI` - future mobile UI exports.
- `10 Dashboard` - dashboard references.
- `11 Mission Control` - Mission Control product assets.
- `12 Sponsor Portal` - Sponsor Portal product assets.
- `13 Photography` - photography direction and future mood boards.
- `14 Illustrations` - illustration direction.
- `15 Backgrounds` - vector texture starters.
- `16 Motion` - motion rules.
- `17 Animations` - Lottie concepts and future exports.
- `18 Data Visualisation` - chart language.
- `19 Social Media` - social template direction.
- `20 Marketing` - campaign and page direction.
- `21 AI` - AI product components.
- `22 Design Tokens` - JSON, CSS, and developer token exports.
- `23 AI Handoff` - prompts for builders and asset generation.
- `24 Asset Manifest` - source inventory and asset status.

## Rule

This system is not decorative. Every asset should help the platform feel more credible, usable, measurable, or commercially valuable.

The full 22-pack map is documented in:

`design-system/24 Asset Manifest/manifests/design-pack-index.md`
